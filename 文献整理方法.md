# 文献整理方法
##  以概念为分类条件

| **应对问题**               | **改进思路**           | **参考文献** |
| ---------------------- | ------------------ | -------- |
| **成本绩效指数的变化**          | 累计统计CPI            |          |
|                        | 概率估计CPI            |          |
|                        | CPI的乐观与悲观估计        |          |
| **早期数据少预测难，对过去信息依赖严重** | 结合多个增长模型           |          |
|                        | 结合GGM增长模型          |          |
|                        | 结合逻辑增长模型           |          |
|                        | 结合GGM增长模型          |          |
| **过往绩效指数同等权重不合理**      | 结合指数平滑技术修改EDI      |          |
|                        | 结合指数平滑技术修改CPI      |          |
|                        | 结合指数平滑技术修改DPI      |          |
| **考虑关键线路与关键活动的影响**     | 引进权重，修改EV，PV       |          |
| **指标测量不准确性**           | 结合模糊理论             |          |
|                        | 结合模糊理论，考虑承包商付款因素影响 |          |
|                        | 应用灰色理论             |          |

## 以论文要素整理

![](file:///C:\Users\zhjj9\AppData\Local\Temp\ksohtml6324\wps1.jpg) 

1. 情境/实践问题/研究问题
2. 文献综述：已有解决方案的构成及不足。
3. 解决方案的理论基础。
4. 研究方案：研究方案的构成。
5. 解决方案的设计：解决方案的构成。
6. 解决方案的评估：评估结果。
7. 结论及对理论与实践的启示。
8. 研究不足及展望

|   |   |   |
|---|---|---|
||**以概念为分类条件**|**以论文要素为分类条件**|
|**内涵**|摘取论文中的概念，进行比较分析|摘取论文的要素|
|**优势**|对概念可进行系统性的梳理；整理的结果可直接用于论文正文|对单篇文章的要素梳理较为完整；对内容的溯源性较好|
|**劣势**|单篇文章的完整性的缺失|需要二次加工整理|
